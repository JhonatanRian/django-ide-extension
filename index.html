Intelligent Context Auto-completion

The extension maps views to their respective templates statically. When you open a template, it parses the view function or class-based view (CBV) that renders it and extracts context variables in real time.

  • Deep model field inspection (e.g. nested objects).
  • Built-in tags and standard Django filters.
  • Custom filter resolution from {% load %}.

Instant View-to-Template Navigation

Eliminate folder exploration friction. The LSP sets up bidirectional links between code references and layouts, enabling direct hyperlinking.

  • Go to Template: Ctrl + Click on a layout path string inside Python code to open it.
  • Go to View: Hit Django: Go to View command to jump back to any Python functions rendering your template.
views.py 
def dashboard(request):
    return render(
        request, 
        "dashboard/main.html", # Ctrl+Click to follow
        {"data": get_stats()}
    )

Database & ORM Intelligence

Write robust Django ORM queries with immediate feedback. The LSP indexes your models, querysets, managers, and migrations files to offer custom-tailored database capabilities.

  • Suggestions for custom Managers, QuerySets methods, and F / Q expressions.
  • Risk assessment analyzer for migrations, preventing production locks or schema corruption.
  • Dynamic fields traversal completion.

Static N+1 Query Prevention

Prevent severe database lag before running server migrations. The static inspector flags potential N+1 database roundtrips occurring during loops inside templates and python views.

  • Identifies loop variables fetching database relations inside HTML templates.
  • Flags nested view loops resolving models' relations.
  • Provides IDE Quick-Fix Actions to add select_related() or prefetch_related().
views.py 
for project in Project.objects.all():
    # Diagnostic alert: N+1 query detected!
    print(project.owner.profile.bio)
⚠️ Django Diagnostic (N+1 Query)
Accessing 'project.owner' inside a loop causes an N+1 query.
💡 Quick Fix: Add select_related('owner')

Built for Speed and Safety

Combining a lightweight TypeScript Client with a blazing-fast Rust Language Server.

🦀

Rust Language Server (LSP)

Written from scratch in Rust, the LSP parser consumes sub-millisecond parsing times. It utilizes highly optimized AST indexing and multithreaded caching so your IDE remains lightning fast.

🌲

Tree-sitter Parsing

Uses tree-sitter grammars to parse both HTML and Python files reliably, tracking accurate scope definitions, context variables, template block bindings, and import mappings.

⚡

Ruff AST Integration

Integrates with ruff_python_ast to efficiently resolve Python syntax trees, ensuring typing patterns, functions returns, context dictionaries, and Class-Based View inheritance are accurately evaluated.

Tailor it to your project

Configure settings directly inside your standard VS Code settings.json.

Setting ID Type Default Description
djangoIde.templateDirs string[] ["**/templates"] A list of glob patterns to search for template folders when scanning.
djangoIde.pythonExecutable string null Path to a custom python interpreter if active virtualenv is not resolved automatically.
djangoIde.diagnostics.n1 boolean true Toggle the static N+1 database Query checker warnings.
djangoIde.diagnostics.migrations boolean true Toggle static risk and safety reviews on migration files.
