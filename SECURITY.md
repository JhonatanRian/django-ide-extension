# Security Policy

## Reporting a Vulnerability

We take the security of our extension and LSP server seriously. If you find a security vulnerability, please do not report it via a public GitHub issue. Instead, report it confidentially.

To report a vulnerability:
1. Please write a detailed email describing the vulnerability, including step-by-step instructions or proof-of-concept code to reproduce the issue.
2. Send your report to the authors or maintainers directly.
3. We will acknowledge receipt of your report within 48 hours and work with you to coordinate a security release to resolve the issue as quickly as possible.
